# Data

## Committed samples

`sample/scene_64.npz` is a synthetic scene generated by this repository's own
simulator: a 64x64 grid holding a smooth random property field (unit std,
correlation length 10 px) with 8 point defects (amplitude 5x the background
std, sigma 2 px) and a measurement-noise setting of 0.3.
`sample/scene_smooth_64.npz` is its defect-free twin: the same seed produces
the identical background with no defects, so the demo can score mapping on
the clean field and defect search on the defect field of the same sample.
Both carry the exact ground truth (noiseless field, background component,
defect centres) plus the generating parameters, and a regression test
confirms each regenerates bit-for-bit from those parameters. They were
created with:

```
activescan simulate --grid 64 --length-scale 10 --defects 8 --noise 0.3 --seed 42 \
    --out data/sample/scene_64.npz
activescan simulate --grid 64 --length-scale 10 --defects 0 --noise 0.3 --seed 42 \
    --out data/sample/scene_smooth_64.npz
```

Nothing in this file comes from any external dataset. It is synthetic and
MIT-licensed with the rest of the repository.

## Bring your own data

No experimental map is committed: openly licensed, clearly attributable
scanning-microscopy property maps are rare, and this repository does not ship
anything with an ambiguous licence. Instead, `activescan.io.load_external`
wraps any fully acquired 2D map of yours as a replayable scene:

```python
from activescan import load_external, run_strategy, gp_reconstruct, rmse

scene = load_external("my_map.npy", noise_sigma=0.0)  # 0 replays values exactly
run = run_strategy(scene, "active_variance", budget=300, seed=0)
recon, _ = gp_reconstruct(scene, run.order, run.values)
print(rmse(recon, scene.field))
```

or from the command line:

```
activescan replay my_map.npy --strategy active_variance --budget 300 --figure replay.png
```

The map acts as the measurement oracle, which answers the practical question:
how much of my own already-acquired scan would an active strategy have needed
to reproduce the map to a given accuracy? Requirements and caveats:

- The array must be square and 2D (`.npy`, or an `.npz` containing exactly one
  array). Crop or resample before loading. Maps are normalised to zero mean
  and unit std so noise settings and thresholds mean the same as in
  simulation.
- Converting vendor formats: load with HyperSpy (`hs.load(...).data`), h5py,
  or tifffile, reduce to a single 2D channel, and `np.save` it.
- A replayed map already contains the noise of the original acquisition, so
  `noise_sigma=0` is the honest default; a nonzero value simulates re-visiting
  with a noisier, faster dwell.
- Replayed scenes have no defect ground truth, so only reconstruction metrics
  apply. If you know defect locations, score hits yourself against
  `run.order`.
